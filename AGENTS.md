# AGENTS.md

This repository is built to survive context compression and fresh agent sessions.

## Rules for future agents
- Read `overview.md` and `architecture.md` before making changes.
- Keep documentation updated when architecture, module boundaries, or workflows change.
- If you spend time debugging a bug or discover an important project detail, record it in `local_notes.md`.
- After completing a meaningful change, add a short summary to `PROJECT_LOG.md`.
- Prefer small, focused changes that match the current architecture.
- Do not invent new subsystems unless they are required by the project goals.

## Documentation files
- `overview.md` — high-level summary of the project.
- `architecture.md` — repo layout and component responsibilities.
- `local_notes.md` — temporary findings, bugs, and working notes.
- `PROJECT_LOG.md` — concise history of completed iterations.
